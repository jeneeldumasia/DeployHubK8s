# Screenshots Checklist

Use this checklist when capturing portfolio screenshots for DeployHub.

## Dashboard

- Project list populated
- Selected project detail panel visible
- Status badge visible
- Action buttons visible

## Live Build

- Build logs actively updating
- Stream state visible
- Project status in `queued` or `building`

## Running Project

- `running` badge visible
- Service URL visible
- Assigned port visible
- Open app button visible

## Error Handling

- Failed project state
- Short error shown in detail panel
- Detailed logs visible in build logs

## Lifecycle Controls

- Stop confirmation prompt
- Delete confirmation prompt
- Redeploy flow visible

## Observability

- `/api/system` response screenshot
- `/metrics` output screenshot
